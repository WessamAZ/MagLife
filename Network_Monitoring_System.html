Architecture

Galaxy collects the raw data from multiple interfaces in the mobile core network using multiple probes. Galaxy probes capture the network packets and dissect them in order to extract the useful information for monitoring purposes. In addition, Galaxy probes correlate different packets related to the same session or dialogue, and group them in a single register called CDR.

CDRs contain different relevant attributes that reflects the needed information to be used at the application layer. Galaxy probes send collected transactions in real time to the Galaxy mediation servers. The mediation servers have an important role in global correlation of different transactions, KPIs and alarms generation, and data enrichment by converting numerical values to a human readable form which reflects the meaning of the numerical codes such as the "Country Name" or the "Operator Name" instead of the numerical codes MCC or MNC.

Real-time and history monitoring

Galaxy uses Elastic Search as history database and Kibana as Graphic User Interface (GUI). You can explore generated CDRs up to the current instant, with rich filtering and aggregation capabilities. Data analysis has never been easier and can be applied on the fly to extract hidden information.
Galaxy can process and generate more than one Billion CDRs per day and can be easily scaled up to accommodate your business growth. Historical CDrs data are available back to three months ago and Key Performance Indicators (KPIs) back to three years ago.
